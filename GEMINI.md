# GSD Workflow Guide (Gemini)

This project uses the Get-Shit-Done (GSD) framework for execution.

## Project Context
**Project:** Olfactory Taxonomy System
**Core Value:** Produzir um sistema semântico olfativo normalizado e computacionalmente útil.
**Current Focus:** Phase 12 closed / v2 default active

## Available Workflow Commands

Use these commands to advance the project:

- `/gsd-progress` - View current progress and get recommended next steps
- `/gsd-discuss-phase [N]` - Chat about the upcoming phase before writing plans
- `/gsd-plan-phase [N]` - Create detailed execution plans for the phase
- `/gsd-execute-phase [N]` - Build the plans in parallel waves
- `/gsd-verify-work` - Run UAT against the delivered features
- `/gsd-transition` - Update state, mark completion, and move to next phase

## Key Artifacts
- `.planning/PROJECT.md` - Core project definition and scope boundaries
- `.planning/ROADMAP.md` - Execution order and phase details
- `.planning/REQUIREMENTS.md` - Trackable features
- `.planning/STATE.md` - Current phase and context pointer
