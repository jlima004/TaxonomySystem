## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbitrary graph -> structural validator | Graph data can contain malformed IDs, endpoints, kinds or stats. |
| structurally valid graph -> profile validator | Profile expectations can reject a graph that is internally consistent but not sanctioned. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-07 | Tampering | `validateOlfactoryGraphStructure` | mitigate | Validate schema, duplicate IDs, endpoints, kinds, alias targets, similarity endpoints and internal stats via factories. |
| T-60-08 | Tampering | `validateOlfactoryGraphAgainstProfile` | mitigate | Short-circuit on structural failure and compare profile stats only after reliable structure. |
| T-60-09 | Repudiation | `validateSanctionedV211Graph` | mitigate | Bind sanctioned validation to named profile ID and baseline constants. |
| T-60-09A | Tampering | validator graph ID checks | mitigate | Use `parseGraphId`/type guards from `graph_id.ts` for node IDs and edge endpoint IDs. |
| T-60-SC | Tampering | npm installs | mitigate | No install tasks; no external validation library. |
</threat_model>
