<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curated seed/corpus analysis → inference functions | Inputs may contain malformed descriptors, very frequent generic terms, or corpus evidence that conflicts with curated seed truth. |
| sparse co-occurrence maps → cluster builder | Pair maps can become large; implementation must not materialize dense descriptor matrices. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `buildSeedCorpusProfiles` | mitigate | Preserve seed descriptors as `source: 'seed'`, emit corpus-only descriptors as candidate/inferred outputs, and assert no seed mutation in tests. |
| T-05-02 | Denial of Service | `buildDescriptorClusters` | mitigate | Iterate sparse co-occurrence keys with `decodePairKey`; do not emit dense matrices or all descriptor cross-products. |
| T-05-03 | Tampering | `scoreSemanticNoise` | mitigate | Downweight noise with audit records; seed-owned noise terms keep weight `1` and emit `seed_exception: true`. |
| T-05-SC | Tampering | npm installs | accept | No package installs are planned; zero runtime dependency constraint remains in force. |
</threat_model>
