<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curated relation/accord JSON → scoring calculators | Malformed curated inputs can create invalid scores or hidden hardcoded domain assumptions. |
| score dimension inputs → calculators | Invalid or missing optional dimensions can create out-of-range scores or accidental penalties. |
| corpus-derived evidence → dimension evidence | Corpus evidence must not obscure curated tradition/accord priorities or become unreviewable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | `computeTraditionScore` / `computeAccordCompatibility` | mitigate | Accept curated relation/accord data as explicit inputs, normalize/validate ids, and return `undefined` for missing optional dimensions. |
| T-05-05 | Tampering | dimension calculators | mitigate | Clamp/validate all dimension scores to `[0,1]` and return `undefined` for absent optional curated dimensions. |
| T-05-06 | Information Disclosure | similarity edge evidence | accept | Evidence contains descriptor ids and aggregate scores only; no secrets or PII are processed in Phase 5. |
| T-05-07 | Tampering | dimension calculator outputs | mitigate | Clamp/validate dimension scores to `[0,1]` and return `undefined` for absent optional curated dimensions so Plan 05-03 can renormalize correctly. |
| T-05-SC | Tampering | npm installs | accept | No package installs are planned; Package Legitimacy Audit says no new package gate is required. |
</threat_model>
