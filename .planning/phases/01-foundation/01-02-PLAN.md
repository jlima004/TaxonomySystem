# Plan 01-02: Definição de Type Definitions (Domain Models)

## Objective

Definir todos os domain types do Taxonomy System em TypeScript strict, cobrindo as quatro estruturas fundamentais: corpus input (materiais enriquecidos), taxonomy seed input (hierarquia manual), taxonomy output compilada, e grafo esparso de similaridade. Todos os types serão exportados via barrel export.

## must_haves

1. Types de corpus que mapeiam a estrutura de `enriched_materials.json` (campos olfativos relevantes)
2. Types de seed que representam a hierarquia manual (family → subfamily → descriptors)
3. Types de output da taxonomia compilada (`taxonomy.json`, `descriptor_aliases.json`, `similarity_matrix.json`)
4. Types de similaridade com suporte a multi-dimensional scores
5. Todos os types compilam sem erros com `tsc --noEmit`
6. Barrel export em `src/types/index.ts`

<action>
Criar `src/types/corpus.ts` com os types que representam os materiais do corpus.

Baseado no schema real do `enriched_materials.json`, definir:

- `CorpusMaterial` — type principal representando um material do dataset
  - `id`: string (formato `cas:NNNNN`)
  - `identity`: `{ name: string, canonical_name: string, aliases: readonly string[], identifiers: { cas?: string, einecs?: string } }`
  - `classification`: `{ category: string, category_path: readonly string[] }`
  - `olfactory`: `OlfactoryProfile` (ver abaixo)
  - `usage`: `{ found_in_nature: readonly string[], uses: readonly string[], blenders: readonly string[] }`
  - `molecular`: `MolecularProperties` (opcionais — match com engine existente)

- `OlfactoryProfile` — dados olfativos de um material
  - `descriptors`: readonly string[] (descriptors brutos do corpus)
  - `primary_type`: string (tipo principal)
  - `odor_description`: string (descrição textual de odor)
  - `descriptor_sources`: `{ tgsc?: readonly string[], sf?: readonly string[] }`

- `MolecularProperties` — propriedades moleculares (parcial, opcional)
  - `cid?`: number
  - `smiles?`: string
  - `molecular_weight?`: number
  - `xlogp?`: number
  - `tpsa?`: number
  - `rotatable_bonds?`: number

Convenções:
- Sem semicolons
- `type` (NÃO `interface`)
- Properties de tipo: snake_case
- Nomes de tipos: PascalCase
- `import type` para type-only imports
- `readonly` em arrays para garantir imutabilidade
</action>

- `CompiledTaxonomy` — output principal (`taxonomy.json`)
  - `version`: string (semver)
  - `generated_at`: string (ISO date)
  - `stats`: `TaxonomyStats`
  - `families`: readonly TaxonomyFamily[]

- `TaxonomyStats`
  - `family_count`: number
  - `subfamily_count`: number
  - `descriptor_count`: number

  - `id`: string (snake_case)
  - `name`: string
  - `subfamilies`: readonly TaxonomySubfamily[]

- `TaxonomySubfamily`
  - `id`: string (snake_case — ex: `floral_white`)

  - `id`: string (normalized lowercase — ex: `jasmine`)
  - `source`: `'seed' | 'corpus' | 'inferred'` (de onde veio)
  - `frequency`: number (contagem no corpus — 0 se não presente)

- `DescriptorAliasMap` — output (`descriptor_aliases.json`)
  - `Record<string, string>` — mapa de alias → canonical ID
  - Type wrapper: `type DescriptorAliasMap = Readonly<Record<string, string>>`

Convenções: mesmo padrão.

<action>
Criar `src/types/similarity.ts` com os types para o grafo esparso de similaridade.

- `SimilarityGraph` — output principal (`similarity_matrix.json`)
  - `version`: string
  - `generated_at`: string
  - `threshold`: number (mínimo para inclusão — default 0.25)
  - `dimensions`: readonly SimilarityDimension[]
  - `edges`: readonly SimilarityEdge[]
  - `stats`: `SimilarityStats`

- `SimilarityDimension` — descreve uma dimensão de similaridade
  - `id`: string (ex: `semantic`, `accord`, `tradition`, `overlap`)
  - `name`: string (display name)
  - `weight`: number (peso na composição — soma = 1.0)

- `SimilarityEdge` — uma aresta do grafo esparso
  - `source`: string (subfamily ID)
  - `target`: string (subfamily ID)
  - `score`: number (composite score — 0.0 a 1.0)
  - `dimensions`: Readonly<Record<string, number>> (score por dimensão)

- `SimilarityStats`
  - `subfamily_count`: number
  - `edge_count`: number
  - `density`: number (edges / max_possible_edges)

Formato de chave `source::target` (pares ordenados alphabeticamente para evitar duplicatas: `floral_white::woody_dry`, nunca `woody_dry::floral_white`).

Convenções: mesmo padrão.
</action>

## Notes

- Os types de `corpus.ts` são baseados no schema real do `enriched_materials.json` — verificados contra os dados reais
- `CanonicalDescriptor.source` distingue de onde cada descriptor veio (seed manual, corpus extraído, ou inferido pelo sistema)
- `SimilarityEdge` usa pares ordenados (`source < target` alphabeticamente) para evitar duplicatas
- Arrays são `readonly` para manter a disciplina funcional do projeto (sem mutação)
- Nenhuma dependency de runtime — os types são puramente compile-time
