# Phase 31 Plan

## Objetivo
Planejar a execução da adição do target (`add_target`) `rosewood` em `taxonomy-seed.v2.json`, sem resolver seus aliases pendentes ou compilar o projeto.

## Escopo Estrito

**O que DEVE ser executado:**
1. Adicionar `rosewood` na family: `woody`, subfamily: `woody_dry` dentro do arquivo `data/taxonomy/taxonomy-seed.v2.json`.
2. Validar que a adição não quebra o schema da taxonomia rodando os testes estáticos (`npm run check:types`, `npm run check:schema`).

**O que NÃO DEVE ser executado ainda (Restrições):**
- **NÃO** alterar `descriptor_aliases.seed.json`.
- **NÃO** criar mapeamentos de aliases para `boi`, `boi_de_rose`, `bois_de_rose` ou `pau_rosa` nesta fase. A resolução dos aliases será realizada exclusivamente na Phase 32.
- **NÃO** rodar o script de compilação da taxonomia (`npm run build` ou similar).
- **NÃO** publicar os artifacts v2 compilados.
- **NÃO** rodar o Graphify (`graphify update .`).

## Passo a Passo da Execução (na Fase Seguinte)

1. **Preflight**: Rodar verificação de git status e garantir que não há alterações pendentes.
2. **Approval**: Exigir a criação de um documento `31-FINAL-APPROVAL.md` explícito antes da mutação.
3. **Mutação Segura**:
   - Abrir `data/taxonomy/taxonomy-seed.v2.json`.
   - Localizar a estrutura `"id": "woody_dry"`.
   - Adicionar `"rosewood"` ao array de `"descriptors"`.
   - Salvar o arquivo.
4. **Validação**:
   - Rodar `npm run check:types` e `npm run check:schema`.
   - Rodar `npm run test` (incluindo `taxonomy_seed_v2.test.ts`).
5. **Closure**:
   - Criar `31-CLOSURE.md` sumarizando a adição bem-sucedida do target.
   - Preparar a passagem para a Phase 32 (resolução de aliases).
