<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw graph -> validated handle | Untrusted or structurally invalid `OlfactoryGraph` attempts to enter query-proof consumption. |
| validated handle -> query consumer | A caller may spoof `ValidatedGraph` with a forced cast or malformed object. |
| query consumer -> low-level query functions | The safe consumer delegates to pure proof generation after validation has succeeded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Spoofing | `ValidatedGraph` handle | mitigate | Use a private runtime brand and profile ID in `query_consumer.ts`; test forced raw graph misuse returns `makeGraphNotValidatedError('validated graph handle required')`. |
| T-61-02 | Tampering | invalid graph structure/profile | mitigate | `asValidatedGraph` calls `validateSanctionedV211Graph` and preserves exact validation errors before any query proof method is reachable. |
| T-61-03 | Repudiation | sanctioned validation profile | mitigate | Store `validation_profile_id: SANCTIONED_V2_11_GRAPH_VALIDATION_PROFILE.profile_id` on successful handles and create them only from the sanctioned wrapper. |
| T-61-SC | Tampering | npm installs | mitigate | No package install tasks; implementation uses existing TypeScript/Vitest only. |
</threat_model>
