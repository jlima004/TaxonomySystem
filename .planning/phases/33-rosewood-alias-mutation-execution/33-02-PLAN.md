# Phase 33-02 — Modern Approval Traceability Integration

## Objetivo
Fazer os testes estruturais reconhecerem approvals modernos em `.planning/phases/*/FINAL-APPROVAL.md`, sem alterar workbooks legados, para permitir que o ciclo de vida do seed evolua para a v2 sem quebrar invariantes de rastreabilidade.

## Escopo
- `taxonomy_seed_v2.test.ts`: Reconhecer `31-FINAL-APPROVAL.md` para suportar `rosewood` (família: `woody`, subfamília: `woody_dry`).
- `alias_seed_v2.test.ts`: Reconhecer `33-FINAL-APPROVAL.md` para suportar o mapeamento de `boi_de_rose` e `bois_de_rose` para `rosewood`.

## Passos de Execução
1. Atualizar `src/tests/curation/taxonomy_seed_v2.test.ts`:
   - Adicionar o path para `31-FINAL-APPROVAL.md` (no diretório de planejamento 31-rosewood-add-target-planning).
   - Fazer o parse dinâmico usando Regex das linhas (Candidato, Família, Subfamília) ou adicionar a entrada na lista manual aprovada para refletir a adição da seed.
2. Atualizar `src/tests/curation/alias_seed_v2.test.ts`:
   - Adicionar a leitura de `33-FINAL-APPROVAL.md`.
   - Fazer o parse das linhas de aliases aprovados (ex: `- boi_de_rose → rosewood`) e incorporá-las ao conjunto de aliases válidos permitidos para os testes passarem sem violar a verificação de legacy workbooks.
3. Rodar os testes isolados:
   `npm run test -- tests/curation/alias_seed_v2.test.ts tests/curation/taxonomy_seed_v2.test.ts`
4. Rodar a suíte completa:
   `npm run test`
5. Atualizar os artefatos de controle:
   - Criar `.planning/phases/33-rosewood-alias-mutation-execution/33-02-SUMMARY.md`
   - Atualizar `.planning/phases/33-rosewood-alias-mutation-execution/33-VALIDATION.md` registrando o sucesso dos testes.
