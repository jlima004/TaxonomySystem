<objective>
Entregar a camada de estatísticas base do corpus normalizado: frequência (document frequency) por descriptor e mapa esparso de co-ocorrência simétrica, expostos por um orquestrador único `analyzeCorpus` que percorre o corpus em uma única passagem. Inclui o módulo de tipos compartilhados (`src/types/analysis.ts`), o serializador determinístico para os artefatos opcionais em `data/analysis/`, e o benchmark CI-safe que detecta regressões severas de complexidade.

Esta plan cobre ANAL-01 (frequência) e ANAL-02 (co-ocorrência) da REQUIREMENTS.md, satisfaz o success criterion 1 do ROADMAP Phase 4 ("matriz de co-ocorrência gerada a partir do corpus"), e estabelece a fundação que a plan 04-02 (similarity + alias candidates) consome via `FrequencyMap`.

**Output:** módulos puros em `src/analyzer/` (zero runtime deps), tipos compartilhados em `src/types/`, fixtures + 5 arquivos de teste em `src/tests/analysis/`.
</objective>

<truths>
**Locked decisions consumidas (cite o ID quando relevante na implementação):**

- **ANAL-D-01** — frequência e co-ocorrência são calculadas EXCLUSIVAMENTE de `material.olfactory.descriptors`. Demais campos textuais (`odor_description`, `descriptor_sources.*`) ficam FORA do escopo.
- **ANAL-D-02** — `identity.aliases` NÃO entra na contagem (metadata de identidade do material, não descriptor olfativo).
- **ANAL-D-03** — Dedup por material ANTES de incrementar frequência: document frequency, não term frequency.
- **ANAL-D-04** — Cada par no mesmo material contribui weight binário +1 após dedup. Sem ponderação.
- **ANAL-D-05** — Pares simétricos armazenados em ordem lex canônica (`a < b`). Half-matrix.
- **ANAL-D-06** — Storage interno: sparse `Map<pair_key, number>`. PROIBIDO matriz densa N×N. Export como edge list `Array<{ a, b, count }>`.
- **ANAL-D-07** — SEM threshold de frequência durante counting. Filtering é responsabilidade downstream (Phase 5 / 04-02 alias step).
- **ANAL-D-12** — Stopwords semânticos (`note`, `nuance`, `effect`, `type`, `quality`, etc.) NÃO são removidos aqui. Diferido para Phase 5.
- **ANAL-D-13** — APIs retornam estruturas `readonly` em memória. Persistência é OPCIONAL e desacoplada (em `src/analyzer/export.ts`).
- **ANAL-D-14** — Quando persistidos, os artefatos vivem em `data/analysis/` com versão explícita no nome: `descriptor_frequency.v1.json`, `descriptor_cooccurrence.v1.json` (e `alias_candidates.v1.json`, escrito por 04-02).
- **ANAL-D-15** — Exports determinísticos byte-for-byte: descriptors ordenados lex; edges ordenadas por `(a, b)` lex; sem timestamps no payload.
- **ANAL-D-17** — Self-pairs `(A, A)` NUNCA são emitidos. Garantido estruturalmente pelo loop `i < j` sobre o descriptor-set ordenado.
- **ANAL-D-18** — Complexidade linear sobre o corpus para freq + cooc. Single-pass mandatório. Benchmark via `performance.now()` em Vitest com threshold CI-safe (NÃO hard contract).
- **Implícito (Phase 3 D-21)** — `normalizeDescriptor` pode retornar `""`. Strings vazias DEVEM ser filtradas (`canonical.length > 0`) antes de entrar no descriptor-set do material.
- **Phase 3 D-23** — `normalizeDescriptor` é idempotente. Phase 4 normaliza no entry sem assumir que o corpus já está canônico.
- **REQ ANAL-01** — Analyzer conta frequência de cada descriptor normalizado.
- **REQ ANAL-02** — Analyzer constrói matriz de co-ocorrência sparse para descriptors.
- **ROADMAP Success Criterion 1** — "Matriz de co-ocorrência é gerada a partir dos dados do corpus" → satisfeito por `computeCoOccurrence` + `analyzeCorpus`.

**Claude's Discretion resolvida aqui (pair key encoding):**
- Pair key encoding = flat `Map<string, number>` com chave `` `${a}|${b}` `` onde `a < b`. O charset canônico de Phase 3 (`^[a-z0-9_]+$`, D-09/D-20) garante ausência de colisão com `|`. Encapsulado em `src/analyzer/pair_key.ts` para isolar a decisão.

**Open Questions encaminhadas (RESEARCH §Risks & Open Questions):**
- **PMI / NPMI / conditional probabilities** — NÃO expor na Phase 4. Diferido para Phase 5; pode ser implementado lá em cima de `FrequencyMap` + `CoOccurrenceMap` sem retrabalho.
- **Camomile↔chamomile com threshold 0.90 default** — concernente a 04-02, citado aqui apenas como nota de integração: o orquestrador `analyzeCorpus` aqui criado já deve aceitar `options.aliasCandidates` (opt-in) sem materializar o pipeline de alias (04-02 wiring).
</truths>
