  <action>
  Read the full 259-candidate inventory from `44-LOW-SUPPORT-INVENTORY.md`. For each candidate, compute a weighted selection score using the model from D-03:

  **Weighted Evidence Model (D-03):**
  - Evidence priority (35%): Based on `candidate_frequency` and `placement_score`. Higher frequency and placement → higher score. Normalize across all 259 candidates.
  - Semantic clarity (25%): Evaluate whether the candidate name is a clear, unambiguous olfactive descriptor. Penalize compound/composite names (e.g., `meaty_roasted_meaty`), generic non-olfactive terms (e.g., `dark`, `soft`, `new`), food-only terms without olfactive identity (e.g., `broccoli`, `lettuce`).
  - Curation value (20%): Assess whether adding this candidate to the taxonomy would improve descriptive coverage. Favor materials and notes used in perfumery/flavor (e.g., `saffron`, `cardamom`, `orchid`, `freesia`). Penalize terms that are purely culinary, bodily, or environmental without olfactive meaning.
  - Low polysemy/risk (15%): Evaluate ambiguity risk. Penalize candidates with multiple non-olfactive meanings (e.g., `nut` could be food or hardware), candidates likely to be corpus artifacts (appearing only in compound contexts), and candidates with suspicious inferred placements.
  - Batch diversity (5%): Favor candidates that add subfamily coverage diversity to the batch. Penalize when a subfamily is already over-represented.

  **Penalty Rules (D-04, D-05):**
  - Corpus artifact penalty: Candidates appearing only in compound forms or with zero support and zero normalized support get strong negative penalty.
  - Generic term penalty: Terms like `dark`, `soft`, `new`, `cooked`, `baked`, `fried` that describe a process/state rather than a smell.
  - Food/off-note caution: Food candidates may be selected if they are recognizable olfactive notes (e.g., `coffee`, `hazelnut`). Off-notes may be selected in limited numbers. Pure noise excluded.
  - Near-duplicate penalty: Compound variants of existing candidates (e.g., `coffee_roasted_coffee` vs `coffee`, `peanut_roasted_peanut` vs `peanut`).
  - New subfamily penalty: Candidates requiring creation of new taxonomy subfamilies get moderate penalty (not exclusion).

  **Selection Process:**
  1. Score all 259 candidates using the weighted model.
  2. Rank by total weighted score descending.
  3. Apply manual sanity review on the top ~60 candidates. For each reviewed candidate, record the sanity review outcome (pass/fail and brief reason).
  4. Select exactly 40 candidates that pass both score threshold and sanity review.
  5. For each selected candidate, write a selection rationale that MUST explicitly include: (a) why the weighted score qualifies it, and (b) the manual sanity review verdict (e.g., "Sanity: pass — recognized perfumery material with clear olfactive identity"). The sanity review must be a visible, recorded part of the rationale, not merely implied by selection.

  **Not-Selected Summary:**
  For all 219 not-selected candidates, assign exactly one closed reason code from:
  - `insufficient_evidence` — frequency/support too low to justify batch inclusion
  - `high_polysemy` — too many non-olfactive meanings
  - `likely_corpus_artifact` — appears only in compound contexts or has zero support with suspicious placement
  - `generic_non_olfactive` — describes a process, state, or generic quality rather than a smell
  - `needs_external_reference` — requires specialized reference to evaluate
  - `requires_new_taxonomy_structure` — would need new subfamilies not yet established
  - `duplicate_or_near_duplicate` — compound variant of an already-selected or higher-priority candidate
  - `low_current_priority` — valid olfactive term but insufficient evidence priority for this batch
  - `out_of_scope_food_or_offnote` — food/off-note/noise term without sufficient olfactive identity for this batch

  **Artifact Structure (D-06):**
  Write `45-BATCH2-SELECTION.md` with these sections:
  1. **Header** — Phase, date, source reference
  2. **Scope** — Selection boundary (from Phase 44 inventory, exactly 40 candidates)
  3. **Selection Policy** — Summary of D-02 through D-05 rules
  4. **Weighted Evidence Model** — Table of weights and penalty descriptions
  5. **Selected Candidates** — Normalized table with 40 rows: #, Candidate, Inferred Subfamily, Frequency, Weighted Score, Selection Rationale
  6. **Not-Selected Summary** — Grouped by reason code: reason code, count, example candidates
  7. **Not-Selected Full List** — Table of all 219: Candidate, Reason Code
  8. **Selection Summary** — Final counts, subfamily distribution of selected batch, zero-mutation confirmation

  **Zero-Mutation Boundary (D-07):**
  - Only create `45-BATCH2-SELECTION.md` in the phase directory
