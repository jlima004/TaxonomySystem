# Plan 27-01: Ambergris Add Target Execution

## Objetivo
Adicionar `ambergris` como um novo seed descriptor oficial na taxonomy v2, classificando-o em `family = amber_resinous` e `subfamily = amber`, habilitando assim sua disponibilidade como anchor point para curadoria e adição de alias futura, sem disparar mutações em outros locais ou realizar publicações oficiais.

## Etapas do Plano

### Passo 1: Preflight e Geração de Aprovação (Approval Gate)
1. Certificar que o repositório está limpo nos artefatos críticos (seed, aliases e compiled files).
2. O analista (usuário/AI) criará o documento `27-FINAL-APPROVAL.md` atestando permissão unicamente para adicionar `ambergris` ao `taxonomy-seed.v2.json`.

### Passo 2: Adição do Target no Seed v2
1. O analista adicionará a string `"ambergris"` no array da subfamily correspondente em `data/taxonomy/taxonomy-seed.v2.json`.
   - Local exato: `family: amber_resinous`, `subfamily: amber`.
2. Validar que somente `taxonomy-seed.v2.json` foi modificado. (Arquivos `descriptor_aliases.seed.json` e o `data/compiled/` permanecerão intactos).

### Passo 3: Teste de Resiliência
1. Rodar localmente os seguintes testes para validação do schema:
   - `npm run check:types`
   - `npm run check:schema`
   - `npm test` (para que o `taxonomy_seed_v2.test.ts` verifique a aprovação formal associada).
2. Se `taxonomy_seed_v2.test.ts` falhar por rastreabilidade de approval, parar a execução e abrir um plano separado 27-02 para approval traceability test integration.

### Passo 4: Report Final
1. Gerar os outputs da validação evidenciando o status final do guard de `PROTECTED_DIFF` validando a mudança exclusiva do target.
2. Encerrar a fase como completa, gerando o `27-01-SUMMARY.md` e o documento `27-CLOSURE.md`. Nenhuma compilação final, snapshot em Graphify ou publicação da v2.4 ocorrerá.
