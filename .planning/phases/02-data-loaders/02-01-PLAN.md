<objective>
Implementar o carregador e validador do taxonomy seed JSON. O loader lê `taxonomy-seed.json` do disco, aplica validação profunda runtime com mensagens descritivas (path + expected + received), e retorna um `TaxonomySeed` tipado. Inclui criação de um seed fixture para testes e o seed real v1 minimal.
</objective>

### Task 2: Implementar seed_validator.ts com 12 regras de validação

<read_first>
- src/types/seed.ts
- src/loader/types.ts
- .planning/phases/02-data-loaders/02-RESEARCH.md §5 (regras R1-R12)
</read_first>

<action>
Criar `src/loader/seed_validator.ts` com função pura `validate_seed(data: unknown): ValidationResult`:

Regras a implementar (acumular TODOS os erros, não fail-fast):
- R1: `version` é string não-vazia
- R2: `metadata.created_at` é string não-vazia
- R3: `metadata.author` é string não-vazia
- R4: `metadata.description` é string não-vazia
- R5: `families` é array não-vazio
- R6: `families[].id` é snake_case (regex `/^[a-z][a-z0-9_]*$/`), único entre todas as families
- R7: `families[].name` é string não-vazia
- R8: `families[].subfamilies` é array não-vazio para cada family
- R9: `subfamilies[].id` é snake_case, único GLOBAL (entre todas as subfamilies de todas as families)
- R10: `subfamilies[].name` é string não-vazia
- R11: `subfamilies[].descriptors` é array não-vazio de strings não-vazias
- R12: Nenhum descriptor duplicado dentro da mesma subfamily

Helpers internos: `is_snake_case(s: string): boolean`, `is_non_empty_string(v: unknown): v is string`

Mensagens devem incluir field path completo: `families[0].subfamilies[1].id`
</action>

<acceptance_criteria>
- `src/loader/seed_validator.ts` exporta `validate_seed`
- Seed válido retorna `{ ok: true, errors: [] }`
- Seed com `families[0].id = "Floral White"` retorna erro com path `families[0].id`, expected `snake_case string`, received `Floral White`
- Seed com subfamily id duplicado detecta e reporta ambos os paths
- Seed com descriptors vazios reporta path exato: `families[0].subfamilies[0].descriptors`
- Acumula múltiplos erros num único ValidationResult
</acceptance_criteria>

### Task 3: Implementar seed_loader.ts

<read_first>

</read_first>

<action>
Criar `src/loader/seed_loader.ts` com:

```
const load_taxonomy_seed: (path: string) => Promise<TaxonomySeed>
```

Implementação:
1. `await readFile(path, 'utf8')` — com try/catch para erro de arquivo não encontrado
2. `JSON.parse(content)` — com try/catch para JSON inválido
3. `validate_seed(parsed)` — se `!result.ok`, throw com mensagens descritivas
4. Return `parsed as TaxonomySeed`

Erros devem ser classes customizadas ou Error com message descritiva contendo todos os ValidationErrors formatados.

Import: `import { readFile } from 'node:fs/promises'`
</action>

<acceptance_criteria>

- Assinatura: `(path: string) => Promise<TaxonomySeed>`

- JSON malformado: rejeita com mensagem mencionando parse error
- Seed inválido: rejeita com TODAS as ValidationErrors formatadas na mensagem
- Seed válido: resolve com objeto `TaxonomySeed`
</acceptance_criteria>
