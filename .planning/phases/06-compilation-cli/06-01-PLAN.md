# Plan 06-01: Schema Validation dos Outputs

<objective>
Implementar tipos compartilhados do compiler (incluindo `CompiledAliases` wrapper) e validadores de schema para os 3 artefatos compilados: `CompiledTaxonomy`, `CompiledAliases` e `SimilarityGraph`. Os validadores são funções puras que recebem `unknown` e retornam `CompilerValidationResult` com erros estruturados.
</objective>

<must_haves>
## Goal-Backward Verification

truths:
  - Validadores retornam structured errors com artifact name, JSON path, error code e message
  - No `null` nos outputs a menos que o schema explicitamente permita
  - Campos opcionais são omitidos, não serializados como null
  - Zero runtime dependencies — validadores são pure functions TypeScript
  - Seguir exatamente os types existentes em `src/types/taxonomy.ts` e `src/types/similarity.ts`
  - `CompiledAliases` é um wrapper: `{ version, schema_version, generated_at, aliases }`
  - Validadores de `descriptor_aliases.json` usam o formato wrapper (não `Record<string, string>` nu)

must_verify:
  - `tsc --noEmit` compila sem erros
  - `vitest run src/tests/compiler/validate_output.test.ts` passa
  - Payloads válidos retornam `{ ok: true, errors: [] }`
  - Payloads inválidos retornam erros com JSON path correto
</must_haves>

<action>
Criar `src/compiler/types.ts` com:

1. **`CompiledAliases`** — wrapper versionado para `descriptor_aliases.json`:
   ```
   type CompiledAliases = {
     readonly version: string
     readonly schema_version: string
     readonly generated_at: string
     readonly aliases: DescriptorAliasMap
   }
   ```
   Import `DescriptorAliasMap` de `../types/taxonomy.js`.

2. **`CompilerValidationError`** — estende o conceito de `ValidationError` com campos adicionais:
   - `artifact: 'taxonomy' | 'aliases' | 'similarity'`
   - `code: string` — e.g. `MISSING_FIELD`, `INVALID_TYPE`, `INVALID_VALUE`, `DUPLICATE_ID`, `EMPTY_ARRAY`, `INCONSISTENT_STATS`, `INCONSISTENT_SCORE`
   - `path: string` — JSONPath-like com prefixo `$`, e.g. `$.families[0].subfamilies[1].id`, `$.aliases["orange flower"]`, `$.edges[0].final_score`. O path raiz é `$`.
   - `message: string` — descrição legível

3. **`CompilerValidationResult`**:
   ```
   type CompilerValidationResult = {
     readonly ok: boolean
     readonly errors: readonly CompilerValidationError[]
     readonly warnings: readonly CompilerValidationError[]
   }
   ```

4. **Helper `makeCompilerError(artifact, code, path, message): CompilerValidationError`** — `path` deve sempre ter prefixo `$`.

5. **Helper `combineResults(...results: CompilerValidationResult[]): CompilerValidationResult`** — merge múltiplos resultados, `ok = false` se qualquer um tem erros.

6. **Helper `findNullsDeep(data: unknown, basePath: string): Array<{path: string}>`** — percorre recursivamente objetos e arrays, retornando todas as posições onde `null` é encontrado. Usado pelos validadores para rejeição profunda de null.

7. Export tudo.
</action>

<task id="06-01-T2">
<title>Implementar validadores de schema</title>

<read_first>
- src/compiler/types.ts (types criados em T1)
- src/types/taxonomy.ts (CompiledTaxonomy, TaxonomyFamily, TaxonomySubfamily, CanonicalDescriptor, TaxonomyStats)
- src/types/similarity.ts (SimilarityGraph, SimilarityEdge, SimilarityEdgeEvidence, SimilarityStats, SimilarityDimension)
- src/types/inference.ts (FinalScoreDimensions, ReviewQueueItem)
- src/loader/seed_validator.ts (referência de padrão de validação — isSnakeCase, isNonEmptyString helpers)
</read_first>

<action>
Criar `src/compiler/validate_output.ts` com:

1. **`validateCompiledTaxonomy(data: unknown): CompilerValidationResult`**:
   - `version`: non-empty string, obrigatório
   - `generated_at`: non-empty string, obrigatório
   - `stats`: objeto com `family_count`, `subfamily_count`, `descriptor_count` — todos `number >= 0`
   - `families`: array (pode ser vazio se taxonomia é empty — mas stats devem ser consistentes)
   - Cada family: `id` (non-empty string), `name` (non-empty string), `subfamilies` (array)
   - Cada subfamily: `id` (non-empty string), `name` (non-empty), `family_id` (deve corresponder ao parent family id), `descriptors` (array)
   - Cada descriptor deve validar: `id` (non-empty string), `source` (exatamente `'seed'`, `'corpus'`, ou `'inferred'`), `frequency` (number >= 0), `status` (`'curated' | 'candidate' | 'inferred'`), `review_required` (boolean), `corpus_derived` (boolean)
   - Invariantes de descriptor:
     - `source: 'seed'` exige `status: 'curated'`, `review_required: false`, `corpus_derived: false`
     - `source: 'corpus'` exige `status: 'candidate'`, `review_required: true`, `corpus_derived: true`
     - `source: 'inferred'` exige `status: 'inferred'`, `review_required: true`, `corpus_derived: true`
   - Verificar unicidade global de family ids e subfamily ids
   - Verificar consistência de stats: `family_count == families.length`, `subfamily_count == soma de subfamilies`, `descriptor_count == soma total de descriptors`
   - **Rejeição profunda de `null`**: usar `findNullsDeep(data, '$')` para detectar qualquer `null` em qualquer nível. Cada ocorrência gera `INVALID_TYPE` com o path exato (e.g. `$.families[0].subfamilies[1].descriptors[0].frequency`). Nenhum campo do `CompiledTaxonomy` permite `null` em v1.
   - Todos os paths de erro usam formato JSONPath com prefixo `$`

2. **`validateCompiledAliases(data: unknown): CompilerValidationResult`**:
   - `version`: non-empty string, obrigatório
   - `schema_version`: non-empty string, obrigatório
   - `generated_at`: non-empty string, obrigatório
   - `aliases`: objeto (pode ser `{}` para zero aliases), cada chave e valor devem ser non-empty strings
   - Nenhum alias aponta para si mesmo (chave === valor)
   - **Rejeição profunda de `null`**: usar `findNullsDeep(data, '$')`. Qualquer `null` em qualquer nível gera `INVALID_TYPE`. E.g. alias com valor `null` gera erro em `$.aliases["key"]`.
   - Todos os paths de erro usam formato JSONPath com prefixo `$`

3. **`validateSimilarityGraph(data: unknown): CompilerValidationResult`**:
   - `version`, `generated_at`: non-empty strings
   - `threshold`: number em `[0, 1]`, preferencialmente `> 0` (se `threshold === 0` gera warning, não erro)
   - `dimensions`: array de `{ id: string, name: string, weight: number }`
   - `edges`: array de objetos com `source`, `target` (non-empty strings)
   - **Validação de scores em edges**:
     - `edge.score`: obrigatório, deve ser `number` em `[0, 1]`. Valor `< 0` ou `> 1` gera `INVALID_VALUE` em `$.edges[i].score`.
     - `edge.final_score`: quando presente, deve ser `number` em `[0, 1]`. Valor fora do range gera `INVALID_VALUE` em `$.edges[i].final_score`.
     - Se `edge.final_score` estiver presente e `edge.score !== edge.final_score`, gerar `INCONSISTENT_SCORE` em `$.edges[i]` com mensagem descritiva.
     - Dimension scores em `edge.dimensions`: cada valor numérico deve estar em `[0, 1]`. Valor fora do range gera `INVALID_VALUE` em `$.edges[i].dimensions.<dim_id>`.
   - `review_queue`: array (pode ser vazio)
   - **Validação de stats**:
     - `stats.subfamily_count`, `stats.edge_count`: `number >= 0`
     - `stats.density`: `number` em `[0, 1]`. Valor fora do range gera `INVALID_VALUE` em `$.stats.density`.
     - Verificar `stats.edge_count == edges.length` — inconsistência gera `INCONSISTENT_STATS`.
   - **Rejeição profunda de `null`**: usar `findNullsDeep(data, '$')`. Nenhum campo do `SimilarityGraph` permite `null` em v1. E.g. `evidence: { curated_relation: null }` gera `INVALID_TYPE` em `$.edges[i].evidence.curated_relation`.
   - Todos os paths de erro usam formato JSONPath com prefixo `$`

4. **`validateAllOutputs(taxonomy: unknown, aliases: unknown, similarity: unknown): CompilerValidationResult`** — roda os 3 validadores via `combineResults()`.

5. **Política de campos desconhecidos (v1)**:
   - Validadores são strict para campos obrigatórios, tipos, valores, nulls e invariantes.
   - Unknown fields NÃO falham em v1, para evitar fragilidade durante evolução interna.
   - Os compilers não devem emitir unknown fields, mas o validator não os rejeita.
   - Nota: schema fechado / unknown field rejection (`UNKNOWN_FIELD`) pode ser endurecido em fase futura.

Import `makeCompilerError`, `combineResults`, `CompilerValidationResult` de `./types.js`.
</action>

   - Fixture com version missing → erro `MISSING_FIELD` com path `$.version`
   - Fixture com families como `null` → erro `INVALID_TYPE` com path `$.families`
   - Fixture com family_id duplicado → erro `DUPLICATE_ID` com path `$.families[i].id`
   - Fixture com descriptor `source: 'invalid'` → erro `INVALID_VALUE`

   - Seed descriptor com `status: 'candidate'` falha

   - **Deep null**: fixture com `frequency: null` em descriptor → erro `INVALID_TYPE` com path `$.families[0].subfamilies[0].descriptors[0].frequency`
   - **Unknown fields**: fixture válida com campo extra `{ ..., foo: 'bar' }` → `ok: true` (tolerado em v1)

   - Fixture válida: `{ version: '1.0.0', schema_version: '1', generated_at: '...', aliases: { a: 'b' } }` → `ok: true`
   - Fixture sem `schema_version` → erro `MISSING_FIELD` com path `$.schema_version`
   - Fixture com alias apontando para si mesmo → erro `INVALID_VALUE`
   - **Deep null**: fixture com alias value `null` → erro `INVALID_TYPE` com path `$.aliases["key"]`
   - Fixture com aliases vazio `{}` → `ok: true` (aliases vazios são válidos)
   - **Unknown fields**: fixture válida com campo extra → `ok: true`

3. **Suite `validateSimilarityGraph`**:
   - Fixture válida mínima → `ok: true`
   - Fixture com edge sem source → erro com path `$.edges[0].source`
   - Fixture com `stats.edge_count != edges.length` → erro `INCONSISTENT_STATS`
   - **Score range**: `edge.score = -0.1` → erro `INVALID_VALUE` com path `$.edges[0].score`
   - **Score range**: `edge.score = 1.5` → erro `INVALID_VALUE` com path `$.edges[0].score`
   - **Score consistency**: `edge.score = 0.5, edge.final_score = 0.8` → erro `INCONSISTENT_SCORE` com path `$.edges[0]`
   - **Dimension score range**: `edge.dimensions.cooccurrence = 1.2` → erro `INVALID_VALUE`
   - **Density range**: `stats.density = 1.5` → erro `INVALID_VALUE` com path `$.stats.density`
   - **Deep null**: `evidence: { curated_relation: null }` → erro `INVALID_TYPE` com path `$.edges[0].evidence.curated_relation`
   - **Unknown fields**: fixture válida com campo extra → `ok: true`

4. **Suite `validateAllOutputs`**:
   - Todos válidos → `ok: true`
   - Um inválido → `ok: false` com erros do artefato que falhou
   - Dois inválidos → ambos os conjuntos de erros presentes

5. **Suite `findNullsDeep`** (helper de types.ts):
   - Objeto sem nulls → array vazio
   - `{a: null}` → `[{path: '$.a'}]`
   - Objeto aninhado com null profundo → path correto
   - Array com null → `[{path: '$[i]'}]`
</action>
