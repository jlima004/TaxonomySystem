# Plan 06-01: Schema Validation dos Outputs

<objective>
Implementar tipos compartilhados do compiler (incluindo `CompiledAliases` wrapper) e validadores de schema para os 3 artefatos compilados: `CompiledTaxonomy`, `CompiledAliases` e `SimilarityGraph`. Os validadores são funções puras que recebem `unknown` e retornam `CompilerValidationResult` com erros estruturados.
</objective>

<must_haves>
## Goal-Backward Verification

truths:
  - Validadores retornam structured errors com artifact name, JSON path, error code e message
  - No `null` nos outputs a menos que o schema explicitamente permita
  - Campos opcionais são omitidos, não serializados como null
  - Zero runtime dependencies — validadores são pure functions TypeScript
  - Seguir exatamente os types existentes em `src/types/taxonomy.ts` e `src/types/similarity.ts`
  - `CompiledAliases` é um wrapper: `{ version, schema_version, generated_at, aliases }`
  - Validadores de `descriptor_aliases.json` usam o formato wrapper (não `Record<string, string>` nu)

must_verify:
  - `tsc --noEmit` compila sem erros
  - `vitest run src/tests/compiler/validate_output.test.ts` passa
  - Payloads válidos retornam `{ ok: true, errors: [] }`
  - Payloads inválidos retornam erros com JSON path correto
</must_haves>

<action>
Criar `src/compiler/types.ts` com:

1. **`CompiledAliases`** — wrapper versionado para `descriptor_aliases.json`:
   ```
   type CompiledAliases = {
     readonly version: string
     readonly schema_version: string
     readonly generated_at: string
     readonly aliases: DescriptorAliasMap
   }
   ```
   Import `DescriptorAliasMap` de `../types/taxonomy.js`.

2. **`CompilerValidationError`** — estende o conceito de `ValidationError` com campos adicionais:
   - `artifact: 'taxonomy' | 'aliases' | 'similarity'`
   - `code: string` — e.g. `MISSING_FIELD`, `INVALID_TYPE`, `INVALID_VALUE`, `DUPLICATE_ID`, `EMPTY_ARRAY`, `INCONSISTENT_STATS`
   - `path: string` — JSON path como `families[0].subfamilies[1].id`
   - `message: string` — descrição legível

3. **`CompilerValidationResult`**:
   ```
   type CompilerValidationResult = {
     readonly ok: boolean
     readonly errors: readonly CompilerValidationError[]
     readonly warnings: readonly CompilerValidationError[]
   }
   ```

4. **Helper `makeCompilerError(artifact, code, path, message): CompilerValidationError`**

5. **Helper `combineResults(...results: CompilerValidationResult[]): CompilerValidationResult`** — merge múltiplos resultados, `ok = false` se qualquer um tem erros.

6. Export tudo.
</action>

<task id="06-01-T2">
<title>Implementar validadores de schema</title>

<read_first>
- src/compiler/types.ts (types criados em T1)
- src/types/taxonomy.ts (CompiledTaxonomy, TaxonomyFamily, TaxonomySubfamily, CanonicalDescriptor, TaxonomyStats)

- src/loader/seed_validator.ts (referência de padrão de validação — isSnakeCase, isNonEmptyString helpers)
</read_first>

<action>
Criar `src/compiler/validate_output.ts` com:

1. **`validateCompiledTaxonomy(data: unknown): CompilerValidationResult`**:
   - `version`: non-empty string, obrigatório
   - `generated_at`: non-empty string, obrigatório
   - `stats`: objeto com `family_count`, `subfamily_count`, `descriptor_count` — todos `number >= 0`
   - `families`: array (pode ser vazio se taxonomia é empty — mas stats devem ser consistentes)
   - Cada family: `id` (non-empty string), `name` (non-empty string), `subfamilies` (array)
   - Cada subfamily: `id` (non-empty string), `name` (non-empty), `family_id` (deve corresponder ao parent family id), `descriptors` (array)
   - Cada descriptor: `id` (non-empty), `source` (exatamente `'seed'`, `'corpus'`, ou `'inferred'`), `frequency` (number >= 0)
   - Verificar unicidade global de family ids e subfamily ids
   - Verificar consistência de stats: `family_count == families.length`, `subfamily_count == soma de subfamilies`, `descriptor_count == soma total de descriptors`
   - Rejeitar qualquer campo `null` — campos não presentes devem estar ausentes, não `null`

2. **`validateCompiledAliases(data: unknown): CompilerValidationResult`**:
   - `version`: non-empty string, obrigatório
   - `schema_version`: non-empty string, obrigatório
   - `generated_at`: non-empty string, obrigatório
   - `aliases`: objeto (pode ser `{}` para zero aliases), cada chave e valor devem ser non-empty strings
   - Nenhum alias aponta para si mesmo (chave === valor)
   - Nenhum valor `null` ou `undefined`

3. **`validateSimilarityGraph(data: unknown): CompilerValidationResult`**:
   - `version`, `generated_at`: non-empty strings
   - `threshold`: number > 0
   - `dimensions`: array de `{ id: string, name: string, weight: number }`
   - `edges`: array de objetos com `source`, `target` (non-empty strings), `score` (number)
   - `dimensions` em cada edge: object
   - `review_queue`: array (pode ser vazio)
   - `stats`: `{ subfamily_count, edge_count, density }` — todos number >= 0
   - Verificar `stats.edge_count == edges.length`
   - `final_score` se presente deve ser number

4. **`validateAllOutputs(taxonomy: unknown, aliases: unknown, similarity: unknown): CompilerValidationResult`** — roda os 3 validadores via `combineResults()`.

Import `makeCompilerError`, `combineResults`, `CompilerValidationResult` de `./types.js`.
</action>

<acceptance_criteria>
- `src/compiler/validate_output.ts` exporta `validateCompiledTaxonomy`, `validateCompiledAliases`, `validateSimilarityGraph`, `validateAllOutputs`
- Payload válido de `CompiledTaxonomy` retorna `{ ok: true, errors: [], warnings: [] }`
- Payload com `families: null` retorna erro com `code: 'INVALID_TYPE'` e `path: 'families'`
- Payload com `families[0].id` duplicado retorna erro com `code: 'DUPLICATE_ID'`
- Payload com descriptor `source: 'invalid'` retorna erro com `code: 'INVALID_VALUE'`
- Alias wrapper sem `schema_version` retorna erro `MISSING_FIELD`
- Alias com chave === valor retorna erro `INVALID_VALUE`
- SimilarityGraph com `stats.edge_count` != `edges.length` retorna erro `INCONSISTENT_STATS`
- `tsc --noEmit` compila sem erros
</acceptance_criteria>
</task>

<action>
Criar `src/tests/compiler/validate_output.test.ts` com:

1. **Suite `validateCompiledTaxonomy`**:
   - Fixture válida mínima: 1 family, 1 subfamily, 1 descriptor seed → `ok: true`
   - Fixture com version missing → erro `MISSING_FIELD`
   - Fixture com families como `null` → erro `INVALID_TYPE`
   - Fixture com family_id duplicado → erro `DUPLICATE_ID`
   - Fixture com descriptor `source: 'invalid'` → erro `INVALID_VALUE`
   - Fixture com stats inconsistente (`family_count != families.length`) → erro `INCONSISTENT_STATS`
   - Fixture com campo `null` em posição não permitida → erro

2. **Suite `validateCompiledAliases`** (wrapper format):
   - Fixture válida: `{ version: '1.0.0', schema_version: '1.0.0', generated_at: '...', aliases: { a: 'b' } }` → `ok: true`
   - Fixture sem `schema_version` → erro `MISSING_FIELD`
   - Fixture com alias apontando para si mesmo → erro `INVALID_VALUE`
   - Fixture com valor null → erro `INVALID_TYPE`
   - Fixture com aliases vazio `{}` → `ok: true` (aliases vazios são válidos)

3. **Suite `validateSimilarityGraph`**:
   - Fixture válida mínima → `ok: true`
   - Fixture com edge sem source → erro
   - Fixture com `stats.edge_count != edges.length` → erro `INCONSISTENT_STATS`

4. **Suite `validateAllOutputs`**:
   - Todos válidos → `ok: true`
   - Um inválido → `ok: false` com erros do artefato que falhou
   - Dois inválidos → ambos os conjuntos de erros presentes
</action>
