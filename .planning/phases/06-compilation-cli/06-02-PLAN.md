# Plan 06-02: Geradores JSON e CLI

<objective>
Implementar os 3 geradores de artefatos compilados (`taxonomy.json`, `descriptor_aliases.json`, `similarity_matrix.json`), o orquestrador all-or-nothing que valida antes de escrever, e o CLI entry point executável via `npm run compile`. Este plano entrega o produto final do milestone v1.
</objective>

<must_haves>
## Goal-Backward Verification

truths:
  - `taxonomy.json` inclui seed descriptors (source='seed') e corpus descriptors (source='corpus') sem promover corpus a curated
  - `descriptor_aliases.json` contém apenas aliases curados do seed, nunca corpus candidates
  - `similarity_matrix.json` preserva edges, dimensions, evidence, review_queue e stats do SimilarityGraph
  - All-or-nothing: se qualquer validação falha, nenhum arquivo é escrito e exit code ≠ 0
  - Output determinístico: sorting estável, generated_at injectable, JSON pretty-printed com trailing newline
  - Zero runtime dependencies
  - CLI funciona via `npm run compile` com defaults e aceita overrides via flags

must_verify:
  - `npm run compile` gera 3 arquivos em `data/compiled/v1/`
  - `tsc --noEmit` compila sem erros
  - `vitest run` — todos os testes passam
  - Duas execuções com `--generated-at` fixo produzem output byte-idêntico
</must_haves>

<title>Compilador de taxonomy.json</title>

<read_first>

- src/types/seed.ts (TaxonomySeed, TaxonomySeedFamily, TaxonomySeedSubfamily)

- src/inference/build_similarity_graph.ts (padrão de buildSubfamilyProfiles e normalizeDescriptor usage)
- src/normalizer/normalize_descriptor.ts (normalize function)
</read_first>

<action>
Criar `src/compiler/compile_taxonomy.ts` com:

1. **Tipo `CompileTaxonomyOptions`**: `{ version: string, generatedAt?: string }`

2. **`compileTaxonomy(seed: TaxonomySeed, analysis: CorpusAnalysis, options: CompileTaxonomyOptions): CompiledTaxonomy`**:
   - Iterar `seed.families` → para cada family e subfamily
   - Para cada descriptor na subfamily seed: criar `CanonicalDescriptor` com `id = normalizeDescriptor(descriptor)`, `source = 'seed'`, `frequency = analysis.frequency.get(id) ?? 0`
   - Coletar descriptors do corpus que pertencem a esta subfamília (via co-ocorrência com descriptors seed) como `source = 'corpus'` — OU simplesmente não adicionar corpus descriptors neste v1 para manter a taxonomia limpa conforme decisão COMP-D-02 (corpus/inferred são materializable mas não obrigatórios no v1 mínimo)
   - **Decisão simplificada v1**: No MVP, `taxonomy.json` materializa APENAS os seed descriptors com suas frequências corpus. Corpus descriptors NÃO são adicionados automaticamente — essa promoção requer curadoria manual. Isso respeita COMP-D-02 estritamente.
   - Sorting: families por `id`, subfamilies por `id`, descriptors por `id`
   - Calcular `TaxonomyStats`: `family_count`, `subfamily_count`, `descriptor_count` (total de descriptors)
   - `version` e `generated_at` dos options (default `generated_at = '1970-01-01T00:00:00.000Z'`)

3. Exportar `compileTaxonomy` e `CompileTaxonomyOptions`.
</action>

<acceptance_criteria>
- `compileTaxonomy()` recebe `TaxonomySeed` + `CorpusAnalysis` e retorna `CompiledTaxonomy`
- Descriptors da seed aparecem com `source: 'seed'` e frequency do corpus
- `stats.family_count === families.length`
- `stats.subfamily_count` conta todas as subfamílias
- `stats.descriptor_count` conta todos os descriptors
- Families e subfamilies estão sorted por id
- Descriptors estão sorted por id
- `generated_at` usa default quando não fornecido
- Nenhum campo `null` no output
</acceptance_criteria>
</task>

</read_first>

<action>
Criar `src/compiler/compile_all.ts` com:

1. **Tipo `CompileAllInputs`**:
   ```
   {
     seed: TaxonomySeed
     aliasSeed: DescriptorAliasSeed
     analysis: CorpusAnalysis
     graphInputs: BuildSimilarityGraphInputs
   }
   ```

2. **Tipo `CompileAllOptions`**:
   ```
   {
     version?: string         // default '1.0.0'
     generatedAt?: string     // default new Date().toISOString()
     outputDir?: string       // default 'data/compiled/v1'
     threshold?: number       // similarity threshold, default 0.25
   }
   ```

3. **Tipo `CompileAllResult`**:
   ```
   {
     ok: boolean
     taxonomy: CompiledTaxonomy
     aliases: CompiledAliases
     similarity: SimilarityGraph
     validation: CompilerValidationResult
     filesWritten?: string[]
   }
   ```

4. **`compileAll(inputs: CompileAllInputs, options?: CompileAllOptions): CompileAllResult`** (puro, sem I/O):
   - Compilar taxonomy via `compileTaxonomy(inputs.seed, inputs.analysis, { version, generatedAt })`
   - Compilar aliases via `compileAliases(inputs.aliasSeed, { version, generatedAt })`
   - Compilar similarity via `buildSimilarityGraph(inputs.seed, inputs.analysis, inputs.graphInputs, { threshold, generatedAt })`
   - Validar com `validateAllOutputs(taxonomy, aliases, similarity)`
   - Retornar resultado com payloads e status de validação
   - NÃO escrever no filesystem — é puro

5. **`writeCompileResults(result: CompileAllResult, outputDir: string): Promise<string[]>`** (I/O boundary):
   - Se `result.ok === false`: throw Error com resumo dos erros de validação
   - Se `result.ok === true`: escrever os 3 arquivos usando `writeJsonDeterministic()` pattern:
     - `${outputDir}/taxonomy.json`
     - `${outputDir}/descriptor_aliases.json`
     - `${outputDir}/similarity_matrix.json`
   - Retornar array de paths escritos
   - Criar dir recursivamente via `mkdir(outputDir, { recursive: true })`
   - Pattern: escrever em temp files primeiro e renomear? Ou escrever diretamente após validação (validação já é all-or-nothing antes da escrita)?
   - Decisão: escrita direta após validação é suficiente para v1 — a validação garante que os payloads são válidos antes de qualquer I/O

6. Exportar tudo.

Atualizar `src/compiler/index.ts`: adicionar exports de `compile_taxonomy.js`, `compile_aliases.js`, `compile_all.js`.
</action>

<acceptance_criteria>
- `compileAll()` é uma função pura que retorna resultado sem side effects
- `writeCompileResults()` lança erro se `result.ok === false`
- `writeCompileResults()` escreve 3 arquivos no diretório especificado
- Cada arquivo JSON é pretty-printed com 2-space indent e trailing newline
- Diretório de output é criado recursivamente se não existir
- `tsc --noEmit` compila sem erros
</acceptance_criteria>
</task>

- src/inference/build_similarity_graph.ts (BuildSimilarityGraphInputs)
- src/package.json (scripts existentes)
</read_first>

<action>
Criar `src/cli/compile.ts`:

1. **Importar** `parseCompileArgs` do parse_args, todos os loaders, `analyzeCorpus`, `compileAll`, `writeCompileResults`

2. **Função `main()`** (async):
   - Parsear args de `process.argv.slice(2)`
   - Se `args.help`: imprimir help text e `process.exit(0)`
   - Imprimir header: `Taxonomy Compiler v1\n`
   - **Loading phase**: 
     - Imprimir `  Loading inputs...`
     - Carregar seed, aliases, corpus, curated relations, accord map
     - Para relations e accords: usar `readFile` + `JSON.parse` diretamente (estrutura simples)
     - Imprimir contagens após cada load: `  ✓ Seed: N families, M subfamilies`
   - **Analysis phase**:
     - Imprimir `  Analyzing corpus...`
     - Rodar `analyzeCorpus(corpus)` para obter frequency, co-occurrence, alias candidates
     - Imprimir `  ✓ Analysis: N descriptors, M alias candidates`
   - **Compilation phase**:
     - Imprimir `  Compiling...`
     - Montar `CompileAllInputs` com seed, aliasSeed, analysis, graphInputs
     - Rodar `compileAll(inputs, { version, generatedAt, outputDir })`
     - Se `result.ok === false`: imprimir erros e `process.exit(1)`
     - Imprimir stats de cada artefato
   - **Write phase**:
     - Imprimir `  Writing outputs...`
     - Rodar `writeCompileResults(result, args.outputDir)`
     - Imprimir paths escritos
   - **Summary**:
     - Imprimir resumo conciso conforme COMP-D-09
     - `process.exit(0)`

3. **Error handling**: wrap `main()` em try/catch global que printa erro e faz `process.exit(1)`

4. **Self-invoke**: `main()` no final do arquivo

5. **Atualizar `src/package.json`**: adicionar script `"compile": "npx tsx cli/compile.ts"`
</action>

<acceptance_criteria>

- Se validação falha, CLI imprime erros e faz exit(1) sem escrever arquivos
- Se validação passa, 3 arquivos são escritos em `data/compiled/v1/`:
