# Plan 06-02: Geradores JSON e CLI

<objective>
Implementar os 3 geradores de artefatos compilados (`taxonomy.json`, `descriptor_aliases.json`, `similarity_matrix.json`), o orquestrador all-or-nothing que valida antes de escrever, a estratégia de escrita temp-file + rename, e o CLI entry point executável via `npm run compile` usando JS compilado (não tsx). Este plano entrega o produto final do milestone v1.
</objective>

<must_haves>
## Goal-Backward Verification

truths:
  - `taxonomy.json` inclui seed descriptors (source='seed') e corpus descriptors (source='corpus') por subfamília, seed primeiro, depois corpus, ambos sorted por id
  - `descriptor_aliases.json` usa wrapper `{ version, schema_version, generated_at, aliases }` e contém apenas aliases curados do seed — nunca corpus candidates
  - `similarity_matrix.json` preserva edges, dimensions, evidence, review_queue e stats do SimilarityGraph
  - `semantic_noise.v1.json` é input explícito do pipeline (passado a `buildSeedCorpusProfiles`)
  - All-or-nothing: se qualquer validação falha, nenhum arquivo é escrito e exit code ≠ 0
  - Escrita via temp-file + rename: `.taxonomy.json.tmp` → `taxonomy.json`
  - Arquivos não relacionados no output dir são preservados (não deletados)
  - Output determinístico: sorting estável, `generated_at` injectable via `--generated-at`, JSON pretty-printed com trailing newline
  - CLI via JS compilado: `npm run build && npm run compile` — sem dependência de `npx tsx`
  - `--generated-at <iso>` flag no CLI; default = current UTC
  - Zero runtime dependencies

must_verify:
  - `npm run build` compila sem erros e emite JS em `dist/`
  - `npm run compile` gera 3 arquivos em `data/compiled/v1/`
  - `vitest run` — todos os testes passam
  - Duas execuções com `--generated-at` fixo produzem output byte-idêntico
  - Validação failure não escreve nenhum arquivo
  - Alias candidates do corpus não aparecem em `descriptor_aliases.json`
  - Arquivos pré-existentes no output dir não são removidos
</must_haves>

<read_first>

</read_first>

<action>
Criar `src/compiler/compile_taxonomy.ts` com:

1. **Tipo `CompileTaxonomyOptions`**: `{ version?: string, generatedAt: string, minCooccurrenceSupport?: number }`
   - `version` default puro: `'1.0.0'`
   - `generatedAt` é **obrigatório** — nenhum default impuro dentro do compiler
   - `minCooccurrenceSupport` default: `1` para v1 simples

2. **`compileTaxonomy(seed: TaxonomySeed, profileResult: SeedCorpusProfileResult, analysis: CorpusAnalysis, options: CompileTaxonomyOptions): CompiledTaxonomy`**:
   - Iterar `seed.families` → para cada family e subfamily:
     - **Seed descriptors**: Cada profile vira `CanonicalDescriptor` com `id = profile.descriptor`, `source = 'seed'`, `frequency = profile.corpus_count`, `status: 'curated'`, `review_required: false`, `corpus_derived: false`.
     - **Corpus descriptors**: Buscar inferred descriptors com suporte `>= (options.minCooccurrenceSupport ?? 1)` com seed descriptors da subfamília. Se tiver suporte com múltiplas subfamilies, escolher a subfamily com **maior suporte**; em empate, escolher o **menor subfamily_id lexicográfico**. Cada um vira `CanonicalDescriptor` com `id = descriptor`, `source = 'corpus'`, `frequency = corpus_count`, `status: 'candidate'`, `review_required: true`, `corpus_derived: true`.
     - **Placement rule**: Na lista `descriptors` de cada `TaxonomySubfamily`, seed descriptors vêm primeiro (sorted por id), depois corpus descriptors (sorted por id).
   - **Inferred descriptors sem atribuição**: descriptors do corpus que não atingem threshold de co-ocorrência com nenhuma subfamília ficam fora do output (não são forçados em lugar nenhum).
   - Sorting: families por `id`, subfamilies por `id`
   - Calcular `TaxonomyStats`: `family_count`, `subfamily_count`, `descriptor_count` (total incluindo corpus)
   - `version` dos options (default `'1.0.0'`), `generated_at` dos options (obrigatório, sem default impuro)

3. Exportar `compileTaxonomy` e `CompileTaxonomyOptions`.
</action>

<acceptance_criteria>
- `compileTaxonomy()` recebe `TaxonomySeed` + `SeedCorpusProfileResult` + `CorpusAnalysis` e retorna `CompiledTaxonomy`
- Seed descriptors têm `status: 'curated'`, `review_required: false`, `corpus_derived: false`
- Corpus descriptors têm `status: 'candidate'`, `review_required: true`, `corpus_derived: true` (e nunca aparecem como curated)
- Descriptor com suporte para duas subfamílias escolhe a de maior suporte; empate escolhe menor `subfamily_id` lexicográfico
- Descriptor sem suporte mínimo fica fora e `minCooccurrenceSupport` é respeitado
- Dentro de cada subfamily: seed descriptors antes de corpus descriptors, cada grupo sorted por id
- `stats.family_count === families.length`
- `stats.subfamily_count` conta todas as subfamílias
- `stats.descriptor_count` conta todos os descriptors (seed + corpus)
- Families e subfamilies sorted por id
- Nenhum campo `null` no output
</acceptance_criteria>
</task>

   ```
   SIMILARITY_FILENAME = 'similarity_matrix.json'
   ```

   - `writeFile(path, content, 'utf8')`

   - Se `result.ok === false`: throw `CompileWriteError` com resumo dos erros de validação
   - Criar dir: `mkdir(outputDir, { recursive: true })`
   - Definir temp paths:
     - `${outputDir}/.${TAXONOMY_FILENAME}.tmp`
     - `${outputDir}/.${ALIASES_FILENAME}.tmp`
     - `${outputDir}/.${SIMILARITY_FILENAME}.tmp`
   - Escrever os 3 payloads nos temp files via `writeJsonDeterministic()`
   - Após os 3 temp files escritos com sucesso, renomear cada um para o nome final via `rename()` (de `node:fs/promises`)
   - Se qualquer write ou rename falhar: cleanup best-effort dos temp files restantes e rethrow o erro (não há transação cross-file perfeita se filesystem falhar durante rename; aceitável para v1)
   - Retornar array de paths finais escritos: `[taxonomy.json, descriptor_aliases.json, similarity_matrix.json]`
   - **Não deletar** arquivos pré-existentes não relacionados no output dir

4. **`CompileWriteError`** — error class customizada com mensagem e erros de validação

5. Exportar `writeCompileResults`, `CompileWriteError`, constantes de filename.
</action>
