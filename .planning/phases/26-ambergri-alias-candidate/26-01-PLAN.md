# Phase 26: Action Plan (Pivot)

## Contexto
O objetivo original da Phase 26 era mapear `ambergri` para `ambergris` como um alias mutation direto. No entanto, o preflight revelou uma falha de invariante crítica: **`ambergris` não existe como um seed válido no sistema**. 

Sendo assim, não podemos executar a mutação de alias, pois isso geraria um ponteiro órfão.

## Estratégia de Pivot
Conforme as regras do sistema, o fluxo deve sofrer um pivot para lidar com a ausência do target. A opção recomendada para este caso (sendo "ambergris" um material clássico de perfumaria essencial para a taxonomia) é a abordagem **1. `add_target` ambergris primeiro**.

### Fase A: Adicionar `ambergris` como Target Seed
1. Identificar a família e subfamília apropriada para "ambergris" (provavelmente Família `animalic`, Subfamília `leathery` ou `musky`, ou uma classificação em `amber_resinous`).
2. Introduzir `ambergris` em `data/taxonomy/taxonomy-seed.v2.json`.

### Fase B: Alias Mutation (Original)
3. Após `ambergris` existir e for validado, introduzir o mapeamento no `data/taxonomy/descriptor_aliases.seed.json`:
```json
{
  "alias": "ambergri",
  "target": "ambergris"
}
```
4. Atualizar os metadados e gerar o diff de estado.

## Próximos Passos (Revisão Necessária)
Antes de procedermos para a Fase A, o usuário deve **aprovar este plano de pivot** e fornecer a direção de classificação (Família/Subfamília) exata sob a qual `ambergris` deve ser incluído no `taxonomy-seed.v2.json` (ou decidir mudar para `defer` / `manual_review_pack`).

**Ações de mutação, compilação ou execução continuam bloqueadas.**
