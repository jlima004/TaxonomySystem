# Phase 21 Plan 01 - Tmp Compile Validation Only

## Objective

Validate that the post-Phase 20 v2.1 inputs can compile cleanly to `/tmp` with explicit paths, fixed version, fixed timestamp, schema validation, and quality gates, without publishing or mutating official artifacts.

## Hard Boundaries

- Do not write to `data/compiled/v2`.
- Do not run bare `npm run compile`, because the default output is official `data/compiled/v2`.
- Do not mutate `data/taxonomy`, `data/inference`, `src/cli/parse_args.ts`, `src/package.json`, `scripts/check-safety-guards.sh`, or `graphify-out/*`.
- Do not commit `/tmp` outputs.

## Tasks

1. Preflight current repo state and record known dirty protected paths without altering them. Existing `graphify-out/*` dirtiness remains out of scope.
2. From `src`, run `npm run build` so `dist/cli/compile.js` reflects current source.
3. Remove any previous `/tmp/taxonomy-phase21-v2-1-compile` directory, then run explicit CLI compile to `/tmp/taxonomy-phase21-v2-1-compile` using:

```bash
node dist/cli/compile.js \
  --seed data/taxonomy/taxonomy-seed.v2.json \
  --aliases data/taxonomy/descriptor_aliases.seed.json \
  --corpus data/enriched_materials.json \
  --relations data/inference/curated_relations.v2.json \
  --accords data/inference/accord_map.v2.json \
  --noise data/inference/semantic_noise.v1.json \
  --out /tmp/taxonomy-phase21-v2-1-compile \
  --version 2.1.0 \
  --generated-at 2026-05-26T00:00:00.000Z \
  --quality-report
```

4. Inspect the three `/tmp` outputs for valid JSON, matching `version: 2.1.0`, matching `generated_at`, expected artifact filenames, quality gate status, and review summary metrics.
5. Compare `/tmp` output behavior against current `data/compiled/v2` at the metric level. Do not require byte equality because `petitgrain` seed promotion and version/timestamp changes are expected.
6. Write `.planning/phases/21-v2-1-compiled-artifact-publication-planning/21-TMP-COMPILE-VALIDATION.md` with command, timestamp, output paths, key metrics, warnings/errors, diff summary, and PASS/FAIL.
7. Verify `data/compiled/v2` has no diff after validation.

## Acceptance Criteria

- `21-TMP-COMPILE-VALIDATION.md` exists and records PASS or FAIL.
- The compile writes only under `/tmp/taxonomy-phase21-v2-1-compile`.
- `/tmp` contains exactly `taxonomy.json`, `descriptor_aliases.json`, and `similarity_matrix.json`.
- All three `/tmp` artifacts use `version: 2.1.0` and the fixed `generated_at` used in the command.
- The compile exits 0 and quality gate status is PASS. If warnings exist, they are listed and classified as accepted or blocking for publication.
- `data/compiled/v2`, `data/compiled/v1`, `data/taxonomy`, `data/inference`, `src/cli/parse_args.ts`, `src/package.json`, `scripts/check-safety-guards.sh`, and `graphify-out/*` are not changed by Plan 21-01.

## Failure Policy

If the `/tmp` compile fails, has hard quality errors, produces missing artifacts, or mutates official files, stop. Do not proceed to Plan 21-02. Record the failure in `21-TMP-COMPILE-VALIDATION.md` and defer official publication.
