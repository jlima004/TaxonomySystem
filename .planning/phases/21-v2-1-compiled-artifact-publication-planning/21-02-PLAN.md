# Phase 21 Plan 02 - Official v2.1 Artifact Publication

## Objective

Publish official compiled v2.1 artifacts to `data/compiled/v2` only after Plan 21-01 records PASS and the user explicitly approves publication. Update CLI default artifact version to `2.1.0` in the same gated publication if approved, while preserving v2 input paths and avoiding any new curation.

## Blocking Prerequisites

- `.planning/phases/21-v2-1-compiled-artifact-publication-planning/21-TMP-COMPILE-VALIDATION.md` exists and records `status: PASS`.
- User explicitly approves official publication and creation of `21-FINAL-APPROVAL.md`.
- The approval artifact records `artifact_version: 2.1.0`, `output_dir: data/compiled/v2`, the fixed `generated_at`, and the exact allowed files.

## Allowlist

- `.planning/phases/21-v2-1-compiled-artifact-publication-planning/21-FINAL-APPROVAL.md`
- `data/compiled/v2/taxonomy.json`
- `data/compiled/v2/descriptor_aliases.json`
- `data/compiled/v2/similarity_matrix.json`
- `src/cli/parse_args.ts` only for `DEFAULT_PATHS.version: '2.1.0'`
- `src/tests/cli/parse_args.test.ts` only for default-version expectation updates
- `src/tests/curation/v1_v2_comparison.test.ts` only for default-version expectation updates

## Blocked Paths

- `data/compiled/v1`
- `data/taxonomy/*`
- `data/inference/*`
- `data/taxonomy/descriptor_aliases.seed.json`
- `scripts/check-safety-guards.sh`
- `src/package.json`
- `graphify-out/*`

## Tasks

1. Confirm Plan 21-01 PASS and inspect `21-TMP-COMPILE-VALIDATION.md` for command, metrics, warnings, and no official diff.
2. Ask the user for explicit official publication approval. If approved, create `21-FINAL-APPROVAL.md` with approval id, artifact version `2.1.0`, fixed `generated_at`, output directory, allowlist, rollback plan, and no-new-curation statement.
3. Update `src/cli/parse_args.ts` default `version` from `2.0.0` to `2.1.0` only after approval. Update targeted tests that assert the default compile version.
4. From `src`, run targeted tests for CLI defaults and v1/v2 comparison expectations.
5. From `src`, run `npm run build`.
6. Run official explicit compile with the approved timestamp and version:

```bash
node dist/cli/compile.js \
  --seed data/taxonomy/taxonomy-seed.v2.json \
  --aliases data/taxonomy/descriptor_aliases.seed.json \
  --corpus data/enriched_materials.json \
  --relations data/inference/curated_relations.v2.json \
  --accords data/inference/accord_map.v2.json \
  --noise data/inference/semantic_noise.v1.json \
  --out data/compiled/v2 \
  --version 2.1.0 \
  --generated-at <approved-release-timestamp> \
  --quality-report
```

7. Inspect official artifacts for valid JSON, version `2.1.0`, matching `generated_at`, and expected quality status.
8. Run final targeted validation and diff audit. The only allowed changes are the approval artifact, three official compiled v2 JSON files, and version-default/test updates listed in the allowlist.

## Acceptance Criteria

- Official publication does not start unless Plan 21-01 PASS and `21-FINAL-APPROVAL.md` exist.
- `data/compiled/v2/taxonomy.json`, `descriptor_aliases.json`, and `similarity_matrix.json` are regenerated from approved v2.1 inputs with version `2.1.0` and the approved fixed `generated_at`.
- `src/cli/parse_args.ts` default version is `2.1.0`; v2 input and output paths remain unchanged.
- No `data/taxonomy`, `data/inference`, `data/compiled/v1`, `scripts/check-safety-guards.sh`, `src/package.json`, or `graphify-out/*` changes occur.
- Targeted tests and typecheck pass, or failures are documented as blockers and official publication is reverted/deferred.

## Rollback

If official publication fails after modifying files, restore the pre-publication versions of the three `data/compiled/v2` JSON files and the CLI/test version updates. Keep `21-FINAL-APPROVAL.md` only if it records the failure and rollback; otherwise remove it before declaring publication deferred.
