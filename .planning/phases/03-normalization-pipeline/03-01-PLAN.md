### Task 1: Criar `normalize_unicode.ts` — NFD decomposition + diacritic removal + ligature expansion

<read_first>

<action>
Criar `src/normalizer/normalize_unicode.ts`:

```
export const normalizeUnicode = (input: string): string => {
  return input
    .normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '')
    .replace(/œ/g, 'oe')
    .replace(/æ/g, 'ae')
    .replace(/Œ/g, 'OE')
    .replace(/Æ/g, 'AE')
}
```

Responsabilidades:
- NFD decomposition (canonical — separa base + combining marks, mais previsível que NFKD)
- Remove combining diacritical marks (U+0300–U+036F)
- Expande ligatures: œ→oe, æ→ae, Œ→OE, Æ→AE
- NFD é preferido sobre NFKD porque é menos agressivo e evita compatibility decomposition inesperada de caracteres que podem ter valor semântico no futuro

NOTA: Esta função NÃO faz lowercase — isso é responsabilidade de `normalizeCase`.
</action>

<acceptance_criteria>
- `normalizeUnicode('frühling')` → `'fruhling'`
- `normalizeUnicode('ñandú')` → `'nandu'`
- `normalizeUnicode('cœur')` → `'coeur'`
- `normalizeUnicode('ÆTHER')` → `'AETHER'`
- `normalizeUnicode('already clean')` → `'already clean'` (idempotente para inputs limpos)
- Não altera case: `normalizeUnicode('ÜBER')` → `'UBER'` (mantém uppercase)
- Idempotency per-step: `normalizeUnicode(normalizeUnicode(x)) === normalizeUnicode(x)`
</acceptance_criteria>

### Task 3: Criar `normalize_separators.ts` — spaces, hyphens, slashes, apostrophes → underscore

<read_first>

- src/loader/seed_validator.ts (isSnakeCase regex)

<action>
Criar `src/normalizer/normalize_separators.ts`:

```
export const normalizeSeparators = (input: string): string => {
  return input.replace(/[\s\-\/\\'\u2018\u2019\u201B\u2013\u2014]+/g, '_')
}
```

Responsabilidades:
- Espaços (whitespace), hyphens (`-`), slashes (`/`), backslashes (`\`) → `_`
- Apostrophe ASCII (`'` U+0027) — funciona como separador lexical em termos como `coeur d'aldehyde` → `coeur_d_aldehyde`
- Typographic apostrophes: left single quotation mark (`'` U+2018), right single quotation mark (`'` U+2019), single high-reversed quotation mark (`‛` U+201B) — comuns em textos copiados de PDFs
- En dash (`–` U+2013) e em dash (`—` U+2014) — usados em textos tipográficos como `fresh–green`
- Múltiplos separators consecutivos colapsam para um único `_` (D-19)
- Esta função é a ÚNICA que introduz underscores

NOTA: O collapse de múltiplos underscores é responsabilidade de `collapseUnderscores`, mas como o regex `[\s\-\/\\'\u2018\u2019\u201B\u2013\u2014]+` já colapsa sequences do mesmo tipo, múltiplos underscores só podem surgir de inputs que já contêm underscores misturados com outros separators (ex: `fresh_-_green`). O `collapseUnderscores` lida com isso.
</action>

<acceptance_criteria>
- `normalizeSeparators('fresh green')` → `'fresh_green'`
- `normalizeSeparators('fresh-green')` → `'fresh_green'`
- `normalizeSeparators('fresh/green')` → `'fresh_green'`
- `normalizeSeparators('fresh---green///woody')` → `'fresh_green_woody'`
- `normalizeSeparators('fresh_-_green')` → `'fresh___green'` (collapse fica para o próximo step)
- `normalizeSeparators('already_snake')` → `'already_snake'`

```
const NON_PLURAL_TERMINALS = new Set([
  'gas',
  'iris',
  'citrus',
  'anis',
  'canvas',
  'chaos',
  'cosmos',
  'ethos',
  'pathos',
  'analysis',
  'thesis',
  'basis',
  'crisis',
  'hypothesis',
  'synthesis',
])

export const singularize = (input: string): string => {
  // Se contém underscores, singularizar cada token individualmente
  if (input.includes('_')) {
    return input
      .split('_')
      .map(singularizeToken)
      .join('_')
  }
  return singularizeToken(input)
}

const singularizeToken = (word: string): string => {
  // Camada 1: dictionary lookup
  const irregular = lookupIrregularPlural(word)
  if (irregular !== undefined) return irregular

  // Camada 1.5: protected non-plural terminals (palavras que terminam em 's' mas NÃO são plurais)
  if (NON_PLURAL_TERMINALS.has(word)) return word

  // Camada 2: suffix stripping rules (ordem importa — mais específico primeiro)
  // ies → y (berries → berry)
  if (word.endsWith('ies') && word.length > 3) return word.slice(0, -3) + 'y'
  // es → apenas após s, x, z, ch, sh (classes → class, boxes → box)
  // NÃO strip genérico de es — quebraria roses → ros, spices → spic
  if (/(ses|xes|zes|ches|shes)$/.test(word)) return word.slice(0, -2)
  // s → remove sufixo s (exceto ss para não quebrar moss → mo)
  if (word.endsWith('s') && !word.endsWith('ss') && word.length > 2) return word.slice(0, -1)

  return word
}
```

Regras de suffix stripping (D-05, corrigidas):
- `ies` → `y` (berries → berry, só se length > 3)
- `es` → `` APENAS após s/x/z/ch/sh (regex contextual — evita roses → ros)
- `s` → `` (exceto palavras terminadas em `ss` como moss, grass)
- Protected terminals: `gas`, `iris`, `citrus`, `anis`, etc. — palavras que terminam em `s` mas são singulares

NOTA: A regra genérica `es` foi removida porque é semanticamente perigosa. Palavras como `roses`, `spices`, `notes` terminam em `es` mas o singular é `rose`, `spice`, `note` (basta remover o `s`). O dictionary-first cobre os irregulares perfumísticos (`mosses`, `leaves`), a regra contextual `es` cobre sibilantes (`classes`, `boxes`), o NON_PLURAL_TERMINALS protege falsos plurais (`gas`, `citrus`), e o fallback `s` cobre o resto.
</action>
