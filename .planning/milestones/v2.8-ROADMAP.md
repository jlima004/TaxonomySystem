## Overview

Continue increasing the curated olfactory descriptor base before MVP work by triaging a second controlled batch from the remaining v2.7 `corpus_candidate_low_support` review queue. v2.8 enforces the same `promote_to_seed` / `add_alias` / `defer` / `reject` / `manual_review` decision lattice as v2.7, but with two non-trivial shifts:

1. **Smaller promotion yield, larger decision surface.** From 259 remaining v2.7 candidates, 40 were selected into Batch 2 and 12 were ultimately promoted to seed descriptors (carrot_seed, freesia, cardamom, tangerine, saffron, osmanthus, cubeb, elderflower, mace, linden_flower, agarwood, tolu). 28 of 40 rows were non-executable with `phase47_instruction=none`.
2. **Closure metrics measured from published JSON.** v2.8 publication (Phase 48) was the first milestone where the closure report's metrics were re-read from `data/compiled/v2/*.json` after the official compile, not from the `/tmp` sandbox validation output. The published-JSON delta vs v2.7 is the canonical source of truth.

## Milestone Summary

**Key Decisions:**

- Batch 2 was fixed at exactly 40 candidates (not the full 25–50 SEL-01 envelope) to match the weighted evidence model's bounded yield.
- Phase 47 receives exactly 12 `promote_to_seed` instructions and zero `add_alias` instructions; all 28 non-executable rows use `phase47_instruction=none`.
- v2.8 publication stays explicit via `--version 2.8.0`; `DEFAULT_PATHS.version` remains `2.1.0`. No default switch.
- Closure metrics are read from the published `data/compiled/v2/*.json`, not from `/tmp` validation output. This becomes the milestone-closure pattern.
- Pre-existing dirty `graphify-out/*` files remain out of scope and are accepted as policy-accepted; protected boundaries are validated via hash assertions + `scripts/check-safety-guards.sh`.
- `cananga` (decision matrix row #4) was deferred to manual review because no `ylang_ylang` seed target existed in the compiled v2.7 taxonomy — this decision row is the explicit binding to the `ylang ylang → ylang_ylang` follow-up below.

**Issues Resolved:**

- 259 remaining v2.7 low_support candidates were inventoried against the current compiled v2.7 review queue.
- 40 candidates were selected with explicit manual sanity-review verdicts and a closed reason-code set covering the 219 not-selected rows.
- Decision matrix enforced closed enums and a strict 12/28 mutation gate; parser verification passed.
- 12 approved seed mutations were applied with full sandbox + Vitest + typecheck + safety-guard validation.
- v2.8.0 compiled artifacts published with version alignment and pre/post deltas vs v2.7.0.
- Closure report produced from published JSON artifacts.

**Issues Deferred:**

- 28 non-executable Batch 2 matrix rows (2 reject, 17 defer_manual_review, 8 defer_future_batch, 1 needs_external_reference) remain in the matrix for future batches.
- 243 remaining `corpus_candidate_low_support` items in the published v2.8 review queue.
- 13 `seed_corpus_conflict` items in the published v2.8 review queue (up from 10 in v2.7.0; conflict count is not part of v2.8's bounded scope).
- MVP, SaaS, Knowledge Engine, UI, Graphify, scoring redesign — explicitly out of scope per the v2.7/v2.8 boundary policy.

**Technical Debt Incurred:**

- **Follow-up for next milestone:** inspect `descriptor_aliases` target integrity, especially `ylang ylang → ylang_ylang`, whose target is not currently present as a compiled taxonomy descriptor. This gap was first surfaced during Phase 8 (`ylang ylang → ylang_ylang` remains as a legacy alias with absent target in the minimal v2 seed) and re-surfaced in the v2.8 Batch 2 decision matrix row #4 (`cananga` deferred because no `ylang_ylang` seed target exists in compiled v2.7). The alias is still being published as-is in the v2.8.0 compiled `descriptor_aliases.json`. The next milestone must decide whether to (a) add a `ylang_ylang` seed target and migrate the alias, (b) drop the alias with rationale, or (c) document a permanent exception. Until then, `data/taxonomy/descriptor_aliases.seed.json` and the published `data/compiled/v2/descriptor_aliases.json` carry a known dangling alias target.
- `graphify-out/*` working tree remains dirty (out of scope by milestone policy). No `data/compiled/v1/` mutation. `DEFAULT_PATHS` not changed.
